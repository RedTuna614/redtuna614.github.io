Descendancy

My capstone project is an isometric squad based tactics game. Within the game the player must take command of four hero mech units and destroy all other hostile mechs and drones. The player must think and use sound tactics when commanding their mechs if they hope to succeed.
With this project my team and I tasked ourselves with making a game prototype, within a couple months, built within Unreal Engine 5.1 using both C++ and Blueprint

Turn Order Sorting algorithm

When the combat manager, the system that controls the turn order, was originally written it stored all units in combat in an unsorted array, and kept track of whose turn it was with an integer called active unit. While this worked, when adding the UI for the turn order, I felt that rather than sorting the unsorted turn order in the UI's blueprint, a new way of tracking the turn order was needed, specifically one that used a sorted array of all units in combat. I thought it would be better to rewrite how the combat manager tracked the order rather than just sorting it in the UI, because this way the turn order would only be tracked in one location, the combat manager, rather than two different places with the UI tracking a sorted turn order and the combat manager tracking an unsorted turn order. Below is the algorithm I wrote to sort and track the turn order in the combat manager. In hindsight, knowing what I do now I'd rewrite the algorithm to use built the functions built into TArrays in Unreal such as sort, pushback, and popback.

Enemy Crusader AI

When making the enemy AI for the game it I originally kept it simple and just had each AI unit target and attack the closest player, or hero unit. If the enemy had a ranged weapon it would move away from the targeted hero unit before attacking, and would just move towards the targeted unit if the enemy either had a melee or the targeted unit was outside their attack range. Later on in an effort to mnake the game more engaging the enemy AI was made more complicated with each unit using a different method to find which player unit it would target. For example, one enemy would prioritize targeting the closest player unit, while another would instead focus on the player unit with the least health remaining. Due to the code's complexity and since each enemy would prioritize a different player unit, the algorithm for finding which player unit to target was written in C++ as an actor component, while simpler methods that each AI used, such as movement, would be written in blueprint.
